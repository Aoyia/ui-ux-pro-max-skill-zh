## v2.0 版本新特性

### 智能设计系统生成

v2.0 的旗舰功能是 **Design System Generator** —— 一个基于 AI 的推理引擎，它可以在几秒钟内分析你的项目需求并生成一个完整的、量身定制的设计系统。


### 设计系统生成的工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 用户请求                                                    │
│     "为我的美容水疗中心构建一个落地页"                          │
│     (Build a landing page for my beauty spa)                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 多领域搜索 (5 个并行搜索)                                   │
│     • 产品类型匹配 (161 个类别)                                 │
│     • 风格推荐 (67 种风格)                                      │
│     • 色板选择 (161 套色板)                                     │
│     • 落地页模式 (24 种模式)                                    │
│     • 字体配对 (57 种字体组合)                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 推理引擎                                                    │
│     • 匹配产品 → UI 类别规则                                    │
│     • 应用风格优先级 (BM25 排序)                                │
│     • 过滤行业反模式                                            │
│     • 处理决策规则 (JSON 条件)                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 输出完整的设计系统                                          │
│     模式 + 风格 + 颜色 + 字体排版 + 效果                        │
│     + 需避免的反模式 + 交付前清单                               │
└─────────────────────────────────────────────────────────────────┘
```

### 161 个行业特定推理规则

推理引擎包含针对以下领域的专业规则：

| 类别 | 示例 |
|---|---|
| **Tech & SaaS (科技与 SaaS)** | SaaS、微型 SaaS、B2B 服务、开发者工具 / IDE、AI / 聊天机器人平台、网络安全平台 |
| **Finance (金融)** | 金融科技 / 加密货币、银行、保险、个人财务记账、发票与账单工具 |
| **Healthcare (医疗健康)** | 医疗诊所、药房、牙科、兽医、心理健康、用药提醒 |
| **E-commerce (电子商务)** | 通用、奢侈品、交易市场 (P2P)、订阅盒、外卖配送 |
| **Services (服务业)** | 美容 / 水疗、餐厅、酒店、法律、家政服务、预订与预约 |
| **Creative (创意行业)** | 作品集、机构、摄影、游戏、音乐流媒体、照片 / 视频编辑器 |
| **Lifestyle (生活方式)** | 习惯追踪、食谱与烹饪、冥想、天气、日记、心情追踪 |
| **Emerging Tech (前沿科技)** | Web3 / NFT、空间计算、量子计算、自主无人机群 |

每个规则包含：
- **Recommended Pattern** —— 落地页结构
- **Style Priority** —— 最佳匹配的 UI 风格
- **Color Mood** —— 适合行业的色板
- **Typography Mood** —— 字体个性匹配
- **Key Effects** —— 动画与交互
- **Anti-Patterns** —— 应当避免的反模式（例如，金融类项目避免使用 “AI 紫色/粉色渐变”）

## 功能特性

- **67 种 UI 风格** —— 玻璃拟态、黏土拟态、极简主义、粗野主义、新拟态、便当格布局、暗黑模式、AI 原生 UI 等
- **161 套色板** —— 与 161 种产品类型 1:1 对应的行业特定色板
- **57 种字体配对** —— 带有 Google Fonts 导入的精选字体组合
- **25 种图表类型** —— 针对仪表盘和分析页面的推荐
- **15 种技术栈** —— React, Next.js, Astro, Vue, Nuxt.js, Nuxt UI, Svelte, SwiftUI, React Native, Flutter, HTML+Tailwind, shadcn/ui, Jetpack Compose, Angular, Laravel
- **99 条 UX 指南** —— 最佳实践、反模式和无障碍性规则
- **161 个推理规则** —— 行业特定的设计系统生成（v2.0 新增）

## 安装指南

### 使用 Claude Marketplace (Claude Code)

在 Claude Code 中直接使用两条命令进行安装：

```
/plugin marketplace add nextlevelbuilder/ui-ux-pro-max-skill
/plugin install ui-ux-pro-max@ui-ux-pro-max-skill
```

### 使用 CLI (推荐)

```bash
# 全局安装 CLI
npm install -g uipro-cli

# 进入你的项目目录
cd /path/to/your/project

# 为你的 AI 助手进行安装
uipro init --ai claude      # Claude Code
uipro init --ai cursor      # Cursor
uipro init --ai windsurf    # Windsurf
uipro init --ai antigravity # Antigravity
uipro init --ai copilot     # GitHub Copilot
uipro init --ai kiro        # Kiro
uipro init --ai codex       # Codex CLI
uipro init --ai qoder       # Qoder
uipro init --ai roocode     # Roo Code
uipro init --ai gemini      # Gemini CLI
uipro init --ai trae        # Trae
uipro init --ai opencode    # OpenCode
uipro init --ai continue    # Continue
uipro init --ai codebuddy   # CodeBuddy
uipro init --ai droid       # Droid (Factory)
uipro init --ai kilocode    # KiloCode
uipro init --ai warp        # Warp
uipro init --ai augment     # Augment
uipro init --ai all         # 所有助手
```

### 全局安装 (适用于所有项目)

```bash
uipro init --ai claude --global   # 安装到 ~/.claude/skills/
uipro init --ai cursor --global   # 安装到 ~/.cursor/skills/
```

### 其他 CLI 命令

```bash
uipro versions              # 列出可用版本
uipro update                # 更新到最新版本
uipro init --offline        # 跳过 GitHub 下载，使用本地打包的资源
uipro uninstall             # 卸载技能 (自动检测平台)
uipro uninstall --ai claude # 卸载指定平台的技能
uipro uninstall --global    # 从全局安装中移除
```

## 使用方法

### 技能模式 (自动激活)

**支持助手：**Claude Code, Cursor, Windsurf, Antigravity, Codex CLI, Continue, Gemini CLI, OpenCode, Qoder, CodeBuddy, Droid (Factory), KiloCode, Warp, Augment

当你请求进行 UI/UX 开发时，该技能会自动激活。只需像平时一样自然地对话即可：

```
Build a landing page for my SaaS product
```

> **Trae**：请先切换到 **SOLO** 模式。该技能将在收到 UI/UX 请求时自动激活。

### 工作流模式 (斜杠命令)

**支持助手：**Kiro, GitHub Copilot, Roo Code, KiloCode

使用斜杠命令来调用该技能：

```
/ui-ux-pro-max Build a landing page for my SaaS product
```

### 示例提示词

```
Build a landing page for my SaaS product

Create a dashboard for healthcare analytics

Design a portfolio website with dark mode

Make a mobile app UI for e-commerce

Build a fintech banking app with dark theme
```

### 工作原理

1. **提出请求** —— 提出任何 UI/UX 任务（构建、设计、创建、实现、评审、修复、优化）
2. **生成设计系统** —— AI 会利用推理引擎自动生成一个完整的设计系统
3. **智能推荐** —— 根据你的产品类型和需求，找到最佳匹配的风格、颜色和字体排版
4. **生成代码** —— 使用正确的颜色、字体、间距和最佳实践来实现 UI
5. **交付前检查** —— 针对常见的 UI/UX 反模式进行校验

### 持久化设计系统 (Master + Overrides 模式)

将你的设计系统保存为文件，以便进行**跨会话的层级检索**：

```bash
# 生成并持久化至 design-system/MASTER.md
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp"

# 同时创建一个特定页面的覆盖文件
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp" --page "dashboard"
```

这会创建如下的 `design-system/` 文件夹结构：

```
design-system/
├── MASTER.md           # 全局单一事实来源 (颜色、字体排版、间距、组件)
└── pages/
    └── dashboard.md    # 特定页面的覆盖 (仅包含与 Master 的差异)
```

**层级检索的工作原理：**
1. 在构建特定页面（例如 "Checkout"）时，首先检查 `design-system/pages/checkout.md`
2. 如果该页面文件存在，其规则将**覆盖 (override)** Master 文件
3. 如果不存在，则排他性地使用 `design-system/MASTER.md`

**上下文感知检索提示词：**
```
I am building the [Page Name] page. Please read design-system/MASTER.md.
Also check if design-system/pages/[page-name].md exists.
If the page file exists, prioritize its rules.
If not, use the Master rules exclusively.
Now, generate the code...
```
