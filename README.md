## v2.0 版本新特性

### 智能设计系统生成器

v2.0 的核心亮点功能是 **Design System Generator**（设计系统生成器）—— 一个基于 AI 的推理引擎，只需几秒钟即可分析您的项目需求，并生成完整且量身定制的设计系统。

### 设计系统生成器底层工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 用户输入需求                                                │
│     "为我的美容水疗中心构建一个落地页"                          │
│     (Build a landing page for my beauty spa)                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 多维度并行检索 (5 个并行搜索)                               │
│     • 匹配产品类型 (161 个类别)                                 │
│     • 设计风格推荐 (67 种风格)                                  │
│     • 色彩搭配选择 (161 套色板)                                 │
│     • 落地页布局模式 (24 种模式)                                │
│     • 字体配对组合 (57 种字体组合)                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 规则推理引擎                                                │
│     • 将产品匹配至 UI 决策规则                                  │
│     • 基于 BM25 算法计算风格优先级                              │
│     • 过滤行业设计反模式/误区                                   │
│     • 解析决策规则（JSON 逻辑判定）                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 输出定制化设计系统                                          │
│     模式 + 风格 + 颜色 + 字体排版 + 效果                        │
│     + 避坑指南（反模式） + 交付前核对清单                       │
└─────────────────────────────────────────────────────────────────┘
```

### 161 条行业特定设计推理规则

推理引擎深度定制了针对以下行业的专业设计规则：

| 行业类别 | 适用细分场景 |
|---|---|
| **Tech & SaaS (科技与 SaaS)** | SaaS、微型 SaaS、B2B 服务、开发者工具 / IDE、AI / 聊天机器人平台、网络安全平台 |
| **Finance (金融)** | 金融科技 / 加密货币、银行、保险、个人理财记账、发票与计费工具 |
| **Healthcare (医疗健康)** | 医疗诊所、在线药房、牙科诊所、宠物医疗、心理咨询与疗愈、用药提醒 |
| **E-commerce (电子商务)** | 通用电商、奢侈品零售、P2P 交易市场、订阅制魔盒、外卖与本地配送 |
| **Services (服务业)** | 美容美体与水疗 (Spa)、餐厅、酒店、法律、家政保洁、在线预约挂号 |
| **Creative (创意行业)** | 个人作品集、设计工作室 / 代理商、摄影展示、独立游戏、音乐流媒体、音视频剪辑工具 |
| **Lifestyle (生活方式)** | 习惯培养与追踪、食谱与烹饪、冥想冥思、天气预报、手帐日记、心情记录 |
| **Emerging Tech (前沿科技)** | Web3 / NFT、空间计算 (VisionOS)、量子计算、无人机群控制 |

每个规则包含：
- **Recommended Pattern** —— 推荐落地页布局
- **Style Priority** —— 最佳匹配的 UI 风格
- **Color Mood** —— 行业调性色板
- **Typography Mood** —— 字体调性匹配
- **Key Effects** —— 核心动效与交互
- **Anti-Patterns** —— 避坑指南/设计反模式（例如：金融类项目应避免使用“俗气的 AI 紫粉渐变”）

## 核心特性

- **67 种 UI 设计风格** —— 磨砂玻璃拟态、黏土拟态、极简主义、新粗野主义、新拟态、便当格布局、OLED 暗黑模式、AI 原生 UI 等
- **161 套行业色板** —— 与 161 种产品类型 1:1 深度契合的行业色板
- **57 组经典字体配对** —— 带有 Google Fonts 导入的精选字体组合
- **25 种图表展现类型** —— 针对仪表盘和分析页面的推荐
- **15 款主流开发框架** —— React, Next.js, Astro, Vue, Nuxt.js, Nuxt UI, Svelte, SwiftUI, React Native, Flutter, HTML+Tailwind, shadcn/ui, Jetpack Compose, Angular, Laravel
- **99 条 UX 交互指南** —— 包含优秀实践、设计反模式与无障碍合规性 (A11y) 规则
- **161 条设计推理规则** —— 行业特定的设计系统智能生成（v2.0 旗舰功能）

## 安装指南

### 通过 Claude 插件市场安装 (Claude Code)

在 Claude Code 终端中直接运行以下两条命令进行安装：

```
/plugin marketplace add nextlevelbuilder/ui-ux-pro-max-skill
/plugin install ui-ux-pro-max@ui-ux-pro-max-skill
```

### 使用 CLI (推荐)

```bash
# 全局安装 CLI
npm install -g uipro-cli

# 进入你的项目目录
cd /path/to/your/project

# 为你的 AI 编码助手安装本地技能：
uipro init --ai claude      # Claude Code
uipro init --ai cursor      # Cursor
uipro init --ai windsurf    # Windsurf
uipro init --ai antigravity # Antigravity
uipro init --ai copilot     # GitHub Copilot
uipro init --ai kiro        # Kiro
uipro init --ai codex       # Codex CLI
uipro init --ai qoder       # Qoder
uipro init --ai roocode     # Roo Code
uipro init --ai gemini      # Gemini CLI
uipro init --ai trae        # Trae
uipro init --ai opencode    # OpenCode
uipro init --ai continue    # Continue
uipro init --ai codebuddy   # CodeBuddy
uipro init --ai droid       # Droid (Factory)
uipro init --ai kilocode    # KiloCode
```

## 设计系统高级 CLI 命令

要直接访问设计系统生成器：

> 注意：如果您是通过 Continue 进行安装的，请在下面的命令中将 `.claude/skills/` 替换为 `.continue/skills/`。对于 Droid (Factory)，请使用 `.factory/skills/`。

```bash
# 生成带有 ASCII 输出的设计系统
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "beauty spa wellness" --design-system -p "Serenity Spa"

# 生成带有 Markdown 输出的设计系统

# 特定设计领域的快速搜索
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "glassmorphism" --domain style
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "elegant serif" --domain typography
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "dashboard" --domain chart

# 特定开发框架的规范检索
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "form validation" --stack react
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "responsive layout" --stack html-tailwind
```

### 持久化设计系统（全局主配置 + 页面覆盖模式）

将设计系统保存为本地文件，供 AI 进行跨会话的层级上下文检索：

```bash
# 生成并保存到全局主配置文件 design-system/MASTER.md
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp"

# 同时生成特定页面的局部覆盖文件
python3 .claude/skills/ui-ux-pro-max/scripts/search.py "SaaS dashboard" --design-system --persist -p "MyApp" --page "dashboard"
```

这将在项目根目录下自动创建如下的 `design-system/` 规范化目录结构：

```
design-system/
├── MASTER.md           # 全局单一事实来源（包含配色、字体、间距、核心组件规范）
└── pages/
    └── dashboard.md    # 特定页面的局部覆盖（仅声明与主配置有差异的特定规则）
```

**层级上下文检索机制：**
1. AI 在构建特定页面（如 Checkout）时，会首先检索 `design-system/pages/checkout.md`
2. 如果该页面配置文件存在，其局部规则将**覆盖 (override)** 全局 `MASTER.md` 的设定
3. 若不存在特定页面配置文件，则默认以 `design-system/MASTER.md` 为全局唯一依据

**上下文感知检索提示词模板 (Context-Aware Prompt)：**
```
I am building the [Page Name] page. Please read design-system/MASTER.md.
Also check if design-system/pages/[page-name].md exists.
If the page file exists, prioritize its rules.
If not, use the Master rules exclusively.
Now, generate the code...
```

## 架构与贡献

### 使用者指南

目前核心代码库已重构为基于模板的动态生成系统。针对各个 AI 助手平台的配置文件（如 `.cursor/`、`.windsurf/`、`.factory/` 等）现统一由 CLI 自动生成并分发。

**建议一律使用 CLI 进行技能的初始化与安装：**

```bash
npm install -g uipro-cli
uipro init --ai <platform>
```

这能确保您正在使用的 AI 助手始终加载最新版本的技能模板与正确的文件层级。

### 贡献者指南

如果您想为该项目做出贡献：

```bash
# 1. 克隆仓库

cd ui-ux-pro-max-skill

# 2. 了解核心目录结构
src/ui-ux-pro-max/           # 单一事实来源（存放 CSV 数据库、构建脚本与发布模板）
cli/                         # CLI 安装程序（从模板生成文件）
.claude/                     # 用于 Claude Code 技能插件的本地调试与测试
```
